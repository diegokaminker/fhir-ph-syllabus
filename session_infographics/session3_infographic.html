FHIR Foundations

FHIR: Fast Healthcare Interoperability Resources

FHIR is the web for healthcare - using modern web technologies to enable healthcare interoperability.

Core Principles

  • Implementer Focus
  • Target the 80% (common scenarios)
  • Use today's web technologies
  • Support human readability
  • Paradigm & architecturally agnostic
  • Open Source

Exchange Mechanisms

  • RESTful API (Primary)
  • Documents (CDA-like)
  • Messages (HL7 V2/V3-like)
  • Services (Operations)
  • Bulk Transfer
  • Subscriptions

Security Checklist

  • Patient Consent
  • Accounting of Disclosure
  • Basic Context (Clock Sync, DNS)
  • Communications (HTTPS, TLS)
  • Integrity (Validation, Provenance)

Why Profiling?

Profiling allows FHIR to be adapted for specific use cases while maintaining interoperability. Everybody knows what to expect, where to find it, and how to get it.
