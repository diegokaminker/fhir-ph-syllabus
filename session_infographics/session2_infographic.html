Healthcare Interoperability

Why Interoperability Matters

Interoperability enables the right information in the right place at the right time for the right person, avoiding data entry redundancy and improving healthcare outcomes.

Interoperability Levels

  • Technical: Information travels between systems
  • Semantic: Receiving system can use the information
  • Process: Exchange improves workflow

Benefits for Healthcare

  • Avoid data entry redundancy
  • Reduce transcription errors
  • Improve care coordination
  • Enable real-time decision support

Benefits for Public Health

  • All health is eventually public health
  • Real-time semantic interoperability
  • Learning health system capabilities
  • Comprehensive population health data

APIs & REST

What is an API?

Application Program Interface - a set of services available to programmers for performing certain tasks. APIs can be internal, partner, or external and serve as contracts between systems.

API Types

  • Internal: Company-only APIs
  • Partner: Company & partner APIs
  • External: Public, open APIs
  • Local: Software libraries
  • Remote: Network-based services

API Evolution

  • EDI (1970s): Text message exchange
  • SOAP (1990s): XML-based services
  • REST (2005+): Resource-based APIs
  • Modern (2012+): gRPC, WebSocket, GraphQL
  • AI (2025+): Agent-to-Agent protocols

API Benefits

  • Access data from third parties
  • Hide complexity
  • Extend functionality
  • Centralize security
  • Standardize data access
